# Задание 3.1 — сортировка строк во вложенных статических массивах

Строки хранятся **только** в структуре

```c
typedef char SmallBlock[SMALL_BLOCK_WIDTH][SMALL_BLOCK_HEIGHT];
typedef SmallBlock LargeBlock[LARGE_BLOCK_WIDTH][LARGE_BLOCK_HEIGHT];
LargeBlock **strings;
```

`strings[i]` — указатель на динамический массив `LargeBlock`, в котором
хранятся символы `SLOTS_PER_GROUP = LARGE_BLOCK_WIDTH * LARGE_BLOCK_HEIGHT`
строк (по одной строке на "слот"). Длина этого массива равна минимально
необходимому числу `LargeBlock` для самой длинной строки данной группы
(с учётом завершающего нуля). Размеры динамических массивов нигде не
хранятся — конец каждой строки определяется только по завершающему `\0`.
Для выделения/освобождения памяти используются исключительно `malloc`
и `free` (без `realloc`).


## Сборка

```
mkdir build
cd build
cmake -S .. -G "Visual Studio 17 2022" -A Win32
cmake --build . --config Release
cmake --build . --config Debug


```

Исполняемый файл появится в `build/app/letnaya_practica_4`
(на Windows — `build/app/Release/letnaya_practica_4.exe` и т.п.).

## Использование

Генерация тестового файла со случайными строками (запрашивает
максимальную длину строки и количество строк):

```
letnaya_practica_4 generate strings.txt
```

Сортировка строк из файла:

```
letnaya_practica_4 sort <input_file> [output_file] [cmp:1-5]
```

Функции сравнения:

| № | Сравнение                                      |
|---|-------------------------------------------------|
| 1 | лексикографически, с учётом регистра (по умолч.) |
| 2 | лексикографически, без учёта регистра            |
| 3 | по длине строки                                  |
| 4 | по последнему символу строки                     |
| 5 | по количеству цифр в строке                      |

Пример:

```
letnaya_practica_4 sort strings.txt sorted.txt 2
```

## Ограничение по вложенности циклов

Доступ к произвольному символу (`GetChar`/`SetChar`) вычисляется по
формулам (деление/остаток) за O(1), без вложенных циклов по всем
измерениям `LargeBlock`/`SmallBlock`. Поэтому нигде в коде не
встречается более двух уровней вложенности циклов.
